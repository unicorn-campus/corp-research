# 프롬프트 예제: 기업 정보조사 (OpenDART + WebSearch 활용)

## 사용 목적
투자 검토·사업 제휴·경쟁사 분석 등 의사결정 전,  
OpenDART MCP로 공시 데이터를 조회하고 WebSearch로 시장 동향을 수집하여  
구조화된 기업 정보조사 레포트 작성 → HTML 대시보드 가시화 → PPT 슬라이드 생성.

---

```text
[목표]
사용자가 입력한 기업명에 대해 OpenDART 공시 데이터와 웹 검색을 활용하여  
기업 현황·재무·시장 포지션을 담은 기업 정보조사 레포트 작성,  
Chart.js 기반 HTML 대시보드 가시화, PPT 슬라이드 생성까지 수행

[역할]
당신은 상장기업 리서치 경력 15년의 증권사 수석 기업분석 애널리스트입니다.

[맥락]
- 내 상황: 투자 검토·사업 제휴·경쟁사 분석을 위해 특정 기업의 종합 정보가 필요한 상황
- 결과물 독자: 사업 기획자·투자 담당자 — 의사결정 근거로 활용할 경영진·실무자

[입력정보]
- 조사 대상 기업명: 사용자에게 질문하여 수집
- OpenDART MCP: 금감원 전자공시 기반 공식 데이터 (재무제표, 주주, 임원, 공시)
- WebSearch: 시장 동향, 경쟁 환경, 최신 뉴스 등 정성 정보
- PPT 스타일 가이드: references/pptx-guide.md
- 출력 기본 경로(`OUTPUT_DIR`): `prompts/3회차/results`

[작업방법]
1. 사용자에게 조사할 기업명 질문
2. 기업 확인 및 기본정보 조회
   - opendart-find_company: 기업명으로 고유번호(corp_code) 검색
   - opendart-get_company_info: 기업 기본정보(업종, 대표자, 설립일 등) 조회
   - 기업이 검색되지 않으면 사용자에게 재입력 요청
3. 레포트 아젠다 초안 생성
   - 아래 기본 아젠다를 기반으로 조사 가능 항목을 표 형식으로 정리
   - 각 섹션별 데이터 소스(OpenDART/WebSearch)를 명시
   - 기본 아젠다:
     | 순서 | 섹션 | 데이터 소스 |
     |------|------|------------|
     | 1 | 기업 개요 | OpenDART |
     | 2 | 재무 분석 | OpenDART |
     | 3 | 주주·임원 현황 | OpenDART |
     | 4 | 최근 공시 동향 | OpenDART |
     | 5 | 시장 포지션·경쟁 환경 | WebSearch |
     | 6 | 최신 뉴스·이슈 | WebSearch |
     | 7 | 종합 평가 | 종합 |
4. **[승인 대기]** 아젠다를 사용자에게 제시하고 수정/승인 요청
   - 사용자가 섹션 추가·삭제·순서 변경 요청 시 → 3번으로 복귀하여 아젠다 수정
   - 사용자가 승인 시 → 5번으로 진행
5. 승인된 아젠다에 따라 섹션별 정보 수집
   - OpenDART 담당 섹션:
     - 기업 개요: opendart-get_company_info, opendart-get_employees
     - 재무 분석: opendart-get_full_financial_statement, opendart-get_financial_index
     - 주주·임원: opendart-get_largest_shareholders, opendart-get_executives, opendart-get_executive_stock
     - 공시 동향: opendart-search_disclosures
     - 배당 정보: opendart-get_dividend_info (아젠다에 포함된 경우)
   - WebSearch 담당 섹션:
     - 시장 포지션: "{기업명} 시장점유율", "{기업명} 경쟁사 비교" 등 검색
     - 최신 뉴스: "{기업명} 최근 뉴스", "{기업명} 이슈" 등 검색
     - 검색 결과는 출처 URL 반드시 병기
6. 수집 데이터를 아젠다 순서대로 레포트 작성
   - 종합 평가: 강점·약점·기회·위협 각 2~3개 도출, 한 줄 총평 포함
7. **[HTML 대시보드 생성]** 레포트 데이터를 Chart.js CDN 기반 HTML 대시보드로 가시화
   - 단일 HTML 파일로 생성 (외부 파일 의존 없이 Chart.js CDN만 사용)
   - 차트 구성:
     - 재무 추이 Bar Chart: 최근 3개년 매출액·영업이익·당기순이익
     - 주주 구성 Doughnut Chart: 최대주주·기관·외국인·기타 지분율
     - SWOT 요약 카드: 강점·약점·기회·위협을 컬러 카드 레이아웃으로 표시
   - 페이지 상단에 기업명·업종·대표자 등 기업 개요 요약 표시
   - 반응형 레이아웃 적용 (모바일에서도 확인 가능)
   - 출력파일: `{OUTPUT_DIR}/{기업명}-dashboard.html`
   - 생성 후 파일 경로를 사용자에게 안내하여 브라우저에서 확인하도록 유도
8. **[PPT 슬라이드 생성]** HTML 대시보드 내용을 기반으로 PPT 생성
   - `references/pptx-guide.md` 스타일 가이드 준수
   - pptxgenjs 기반 build.js 스크립트를 작성하여 node로 실행
   - 슬라이드 구성 (4장):
     - 슬라이드 1: 표지 — 기업명, 조사일, "기업 정보조사 레포트" 제목
     - 슬라이드 2: 기업 개요 + 재무 분석 (패턴 D: 테이블 + 상세 섹션)
       - 기업 기본정보 표 + 최근 3개년 재무지표 표
     - 슬라이드 3: 시장 포지션 + 경쟁 환경 (패턴 B: 다이어그램 + 설명 2열)
       - 시장 내 위치 요약 + 주요 경쟁사 비교
     - 슬라이드 4: SWOT 종합 평가 (패턴 A: 카드 그리드 2x2)
       - 강점·약점·기회·위협 4개 카드 + 하단 총평
   - 빌드 스크립트 규칙:
     - `pptx.defineLayout({ name: "CUSTOM", width: 16, height: 9 })` 사용
     - 최소 폰트 크기 12pt 이상 (fs12 헬퍼 함수 경유)
     - 폰트: Pretendard 통일
     - 도형: `pptx.shapes.*` 상수만 사용
     - 테이블: `slide.addTable()` 사용 (셀 수동 그리기 금지)
     - 각 슬라이드는 `async function createSlideXX(pptx)` 패턴으로 분리
   - 출력파일: `{OUTPUT_DIR}/{기업명}-report.pptx`
   - 빌드 실행: `node {OUTPUT_DIR}/{기업명}-build.js`
   - 생성 후 파일 경로를 사용자에게 안내
- 톤앤매너: 객관적·분석적, 사실과 의견 명확히 구분
- 작성 규칙:
  - OpenDART 데이터는 "(출처: DART 전자공시, YYYY년 N분기)" 형식으로 병기
  - WebSearch 데이터는 출처명·URL·날짜 병기
  - 추정치는 "(추정)" 표기, 출처 불명확 수치는 "정확한 수치 확인 필요" 명시
  - 재무 수치는 억원 단위로 표기, 표 형식 활용
  - 명사체 종결 사용 ("~함", "~임")

[출력]
- 레포트: 응답으로 출력 (마크다운 형식)
- HTML 대시보드: `{OUTPUT_DIR}/{기업명}-dashboard.html`
- PPT 슬라이드: `{OUTPUT_DIR}/{기업명}-report.pptx`
- 출력 구조 (승인된 아젠다 기준, 기본 구성):
  ## 1. 기업 개요
  (기업명, 업종, 대표자, 설립일, 직원수 등)

  ## 2. 재무 분석
  (매출액, 영업이익, 당기순이익 등 주요 재무지표 — 표 형식)

  ## 3. 주주·임원 현황
  (최대주주, 지분율, 주요 임원 — 표 형식)

  ## 4. 최근 공시 동향
  (최근 주요 공시 3~5건 요약)

  ## 5. 시장 포지션·경쟁 환경
  (시장 내 위치, 주요 경쟁사 비교)

  ## 6. 최신 뉴스·이슈
  (최근 주요 뉴스 3~5건 요약, 출처 링크 포함)

  ## 7. 종합 평가
  (강점·약점·기회·위협 + 한 줄 총평)

  ---
  ※ OpenDART 데이터는 금감원 전자공시 기준이며, 최신 공시 반영 시점에 따라 차이가 있을 수 있음.

[제약조건]
- MUST:
  - opendart MCP를 통해 재무제표·기업정보를 1개 이상 실제 조회 후 인용
  - 아젠다 승인 전까지 본격 정보 수집을 시작하지 않음
  - WebSearch 결과는 출처 URL 병기
  - HTML 파일은 외부 파일 의존 없이 Chart.js CDN만 사용하여 단일 파일로 완결
  - PPT 생성은 pptxgenjs 기반 build.js 스크립트로 실행 (references/pptx-guide.md 준수)
- MUST NOT:
  - MCP 조회 없이 기억·추론만으로 재무 수치를 작성하지 말 것
  - 출처 불명확한 수치 단독 사용 금지
  - anthropic-skills:pptx 등 외부 변환 스킬 사용 금지
- 완료조건:
  - 승인된 아젠다의 모든 섹션이 데이터와 함께 출력되고, 종합 평가가 포함됨
  - `{OUTPUT_DIR}/{기업명}-dashboard.html` 파일이 생성되어 브라우저에서 차트가 정상 표시됨
  - `{OUTPUT_DIR}/{기업명}-report.pptx` 파일이 생성되고 0바이트 초과

[예시]
(입력)
기업명: ICTK홀딩스

(아젠다 제시 예시)
```

## 활용 팁
- 기업명만 입력하면 아젠다 제시 → 승인 → 레포트 → HTML 대시보드 → PPT까지 대화형으로 진행
- 아젠다 승인 단계에서 "배당 정보 추가", "뉴스 제외" 등 자유롭게 커스터마이즈 가능
- `[역할]`을 "반도체 산업 전문 애널리스트"로 변경하면 도메인 특화 분석 가능
- 여러 기업을 비교하려면 기업명을 쉼표로 구분하여 입력 (예: "ICTK홀딩스, 삼성전자")
- HTML 대시보드는 `{OUTPUT_DIR}/` 디렉토리에 저장되므로 브라우저에서 직접 열어 확인 가능
- PPT는 pptx-guide.md 스타일이 적용되어 프로젝트 내 다른 교재와 일관된 디자인 유지
